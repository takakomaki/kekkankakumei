## 1. この指示書の目的（何を作る？）

このリポジトリ `kekkankakumei` は、**「加藤先生（薬剤師・医薬研究者）による血管/血流を軸にした“薬に頼り続けない未来”の啓蒙〜セミナー申込**へ繋ぐLP（ランディングページ）を、Next.jsで実装するプロジェクト。

- **最終ゴール（Primary CTA）**: 「加藤先生セミナー」への申込（フォーム/決済/予約導線のいずれか）
- **副次ゴール（Secondary CTA）**: 公式LINE登録 / YouTube視聴 / 書籍「血管革命」への導線

Vercelで本番公開され、GitHubと連携されている前提。

- Vercel: `https://vercel.com/takakomaki-7193s-projects/kekkankakumei`
- GitHub: `https://github.com/takakomaki/kekkankakumei`

## 2. 背景ソース（コピーの一次情報）

`docs/news_souten.md` に、対談「血管革命」に関する書き起こしがある。LP本文の“主張/根拠/注意点（極端な断薬論ではない等）”はここから抽出して、セクションへ割り当てる。

特に終盤で、以下の導線が明記されている:
- 書籍「血管革命」への申込導線（概要欄リンク）
- YouTube / 公式LINE

## 3. 現状（コードがどこまで出来ている？）

LP本体は `app-web/` に存在（Next.js App Router + Tailwind v4）。

- **ページ構成**: `app-web/app/page.tsx` が9セクションを順に描画
- **Heroは完成度高め**: `app-web/app/components/sections/hero-section.tsx`
  - ダーク背景 + 金の血管ライン + 深紅のパルス
  - 明朝系フォント、静謐で高級感のあるトーン
  - コピー例: 「血管は、あなたの未来の地図。」「薬に頼り続ける人生に、静かな違和感を覚えているあなたへ。」
- **Hero以外は骨格/仮文言が多い**: `philosophy/problem/common-solution/true-cause/proof/cta` などはプレースホルダー中心
- **メタデータ未整備**: `app-web/app/layout.tsx` が `title: "Create Next App"` などテンプレのまま、`<html lang="en">` のまま

## 4. 目指すLPの“意図”（メッセージ設計）

### トーン&禁止事項

- **トーン**: 不安煽りの煽動ではなく、静かな危機感→納得→希望→具体策→申込、の順で落とす。Heroの世界観（静謐/上質/医療寄りの信頼）に合わせる。
- **禁止/注意**:
  - 「薬=毒」「医者が悪」などの断定・陰謀論に寄せない
  - `news_souten.md` のスタンス（“薬は良い薬もある”“向き合い方/卒業の設計”）を保つ
  - 医療行為の断定・診断・断薬指示に見える表現は避け、必要なら免責を入れる

### 想定読者（最低限）

- 40〜70代中心（またはその家族）
- 高血圧/脂質異常/血糖など「数値」で不安になりやすい
- 薬が増えていくことに違和感があるが、極端な反医療は望んでいない
- 「病院に行かない体づくり」「根本療法」という言葉に反応する

## 5. セクション別：入れるべき内容の割り当て（重要）

実装ファイルはすでに分割されている。各セクションに入れるべき“論点”を以下に固定し、コピーとUIを作る。

### 5.1 Hero（`hero-section.tsx`）

- 既存の世界観を維持しつつ、**LPの目的（セミナー申込）**に接続する要素を追加
  - 例: 「加藤先生セミナー」ラベル、開催概要（日時/場所/オンライン等）、主要ベネフィット3点、CTAボタン
- CTAは “今すぐ申込” を主、 “まずはLINEで情報を受け取る” を副

### 5.2 Philosophy（`philosophy-section.tsx`）

- “薬に頼らない未来へ” の思想を言語化
- 「病院/薬を否定しないが、最終目標は“薬が要らない状態”」という整理
- 先生の立ち位置: 薬剤師・研究者として“別角度の警鐘”

### 5.3 Problem（`problem-section.tsx`）

- 多くの人が“正しい”と思う対処の盲点
- `news_souten.md` から論点抽出:
  - 血管疾患が主要死因（心疾患/脳血管）
  - 高血圧人口の多さ（隠れ高血圧含む）
  - 血管老化が根なのに「詰まらない/破れない」対症へ寄りがち

### 5.4 Common Solution（`common-solution-section.tsx`）

- よくある解決策（例: “数値を下げることが正義” / 薬でコントロール）を提示し、メリット/限界をフェアに説明
- 2000年頃の基準値の話（“正常値→基準値”など）を、断定しすぎず説明の材料として使う

### 5.5 True Cause（`true-cause-section.tsx`）

- 核心: **血流/酸素・栄養の配送遅延 → 老化/不調が加速**
- “血管は筋肉で、鍛えられる/裏切らない” を中心メッセージに置く
- 簡易セルフチェック案（例: 階段で息が上がる、手足が冷える、立ちくらみ等）※医療断定を避ける

### 5.6 Heaven（`heaven-section.tsx`）

- “血管を鍛える” を続けた未来の描写（既存の仮メモを具体化）
  - 朝の目覚め、検査数値への不安軽減、家族との時間、旅行/趣味

### 5.7 Hell（`hell-section.tsx`）

- 何もしない/対症だけの未来（既存の仮メモを具体化）
  - 薬が増える、不安が消えない、「このままでいいのか」
- ただし過剰な恐怖訴求にしない（静かな危機感）

### 5.8 Proof（`proof-section.tsx`）

- 信頼の根拠:
  - 加藤先生の肩書・経歴（外資製薬で研究、現場のギャップ、ホリスティック等）
  - データ/引用（`news_souten.md` の数値・主張）
  - 「薬は良い薬もある」「卒業の設計」などバランス感
- できれば第三者要素（掲載実績/動画/書籍/レビュー等）をここに集約

### 5.9 Consultation/CTA（`consultation-cta-section.tsx`）

- 本来ここは「セミナー申込」へ最短で落とすセクション
  - 申込ボタン（フォームURL/LINE/予約）
  - 開催概要（日時、場所、参加費、定員、対象者、特典、キャンセル等）
  - よくある質問（薬をやめろと言われる？等の不安払拭）

## 6. 画像/OGP運用（生成フローが既にある）

`docs/gemini_image_workflow.md` に画像生成・配置ルールがある。

- 画像は `app-web/public/generated/` に置き、Next.jsから `/generated/...` で参照する
- 例: `/generated/hero.webp` `/generated/ogp.png`

注意: ドキュメントでは `prompts/images/` を前提としているが、現状リポジトリ内に見当たらないため、必要なら作成すること（運用の一貫性のため）。

## 9. 依頼する作業の粒度（他AIに渡す具体タスク）

### コピー/構成担当AIに渡す

- `docs/news_souten.md` から “LPに必要な主張/根拠/注意点” を抽出し、上記セクション割り当てに沿って文章化
- セミナー固有情報（日時/参加費/会場/申込URL/特典/対象者/注意事項）を、プレースホルダー込みで整形
- FAQを作成（薬/医者否定ではない、断薬指示ではない等）

### 実装担当AIに渡す

- 既存Heroの世界観を踏襲し、残り8セクションを実装（レスポンシブ、読みやすい余白、和文タイポ）
- `layout.tsx` のSEO/OG/言語設定をLP向けに更新
- CTA導線（外部フォーム/LINE）をボタン化（リンクは設定値で差し替え可能に）
- 画像配置規約（`public/generated/`）に従い、Hero/各セクションの画像枠を用意

### 生成画像担当AIに渡す（Gemini/画像）

- `public/generated/` に配置すべき画像セットを定義
  - `hero.webp`（Hero背景/キービジュアルが必要なら）
  - `ogp.png`
  - セクション用（proof/true-cause等）
- 画像プロンプトのファイル体系（`prompts/images/`）を新設し、プロンプトを格納
